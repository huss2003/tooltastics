### 3.2 Cross-Linking Rules

**Every tool page links to:**
- 2 related tools in the content body ("Also try our [Related Tool]")
- 1 related guide in the FAQ ("Learn more in our [Guide]")
- The calibration guide (if applicable)
- The homepage in breadcrumb

**Every guide page links to:**
- The relevant tool page in H2/H3 sections ("Use our [Online Ruler] to...")
- 2 other related guides
- The homepage in breadcrumb

**Homepage links to:**
- All 8 tool pages (via tool grid)
- Top 3 guide pages (via featured content section)
- FAQ page (via FAQ section)
- About, Contact (via footer)

**Static pages link to:**
- Homepage (breadcrumb)
- Related tools (contextually)

## 4. Programmatic SEO Opportunities

### 4.1 Device PPI Database (500+ Pages)

**Concept:** Generate a page for every device with its exact screen specs.

**URL Pattern:** `/en/screen-specs/[brand]/[model]`
- `/en/screen-specs/apple/iphone-15-pro-max`
- `/en/screen-specs/samsung/galaxy-s24-ultra`
- `/en/screen-specs/dell/xps-15-9530`

**Data Source:** The existing device database in `src/lib/calibration/deviceDatabase.ts` (40+ entries, expandable to 500+)

**Template:**
```astro
---
// Generated from device database
const device = getDevice(params.brand, params.model)
---
<h1>{device.name} Screen Specifications</h1>
<p>PPI: {device.ppi} | Resolution: {device.width}x{device.height} | Diagonal: {device.diagonal}"</p>
<!-- Auto-calibration link, ruler tool link, related devices -->
```

**SEO Value:**
- Long-tail: "iphone 15 pro max screen ppi"
- Volume: 20K+/mo across all devices
- Low competition (spec pages are thin content on most sites)
- High CTR (navigational intent)

**Expansion Path:**
1. Start with top 100 devices (phones, tablets, laptops, monitors)
2. Add desktop monitors (expand by brand)
3. Add TV sizes
4. Auto-generate from a structured data file
